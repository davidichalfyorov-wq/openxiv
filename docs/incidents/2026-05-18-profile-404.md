# Incident: ORCID profile 404 in production (Phase 0 audit)

**Date:** 2026-05-18 (audit only, no code changes)
**Reporter:** owner
**Severity:** P1 (single affected user, broken UX surface)

## Symptoms

- Sign-in via ORCID succeeds (David Alfyorov, ORCID 0009-0003-6027-7837)
- Header avatar/name → href `/@did%3Aweb%3Aopenxiv.local%3Aorcid.0009-0003-6027-7837`
- Click navigates to **/profiles/did%253Aweb%253Aopenxiv.local%253Aorcid.0009-0003-6027-7837**
  (three percent-layers: original colons → `%3A` → encoded again to `%253A`)
- Page renders "Profile not available"; API body:
  `{"kind":"not_found","message":"profile did%3Aweb%3Aopenxiv.local%3Aorcid.0009-0003-6027-7837"}`

## Reproduction

```bash
curl -s 'https://openxiv.net/api/profiles/did:web:openxiv.local:orcid.0009-0003-6027-7837' \
  → 200 OK (raw colon path works — Fastify URL parser decoded once)
curl -s 'https://openxiv.net/api/profiles/did%3Aweb%3Aopenxiv.local%3Aorcid.0009-0003-6027-7837' \
  → 200 OK
curl -s 'https://openxiv.net/api/profiles/did%253Aweb%253Aopenxiv.local%253Aorcid.0009-0003-6027-7837' \
  → 404 not_found
```

## Root causes (two, independent)

### RC1 — deployed build is from before the Phase 5 fix

VPS container `openxiv-api-1` image built 2026-05-17 19:40 UTC.
- `/app/apps/api/dist/routes/profiles.js` does NOT contain `normalizeProfileIdentifier`
- No `.js` in `/app/apps/api/dist` references `canonicalDidForProfile`
- `/app/apps/web/dist/server/pages/` still has `@_handle_.astro.mjs` and has **no** `u/` directory

Local repo at `D:\OpenXiv` *does* have the fixes:
- `apps/api/src/routes/profiles.ts` — `normalizeProfileIdentifier` (decode-until-stable, max 5)
- `apps/api/src/routes/profiles.ts` — `canonicalDidVariants` (openxiv.local → openxiv.net:u:)
- `apps/web/src/pages/u/[handle].astro` (replaces `@[handle].astro`)
- `apps/web/src/middleware.ts` + `apps/web/src/lib/url-salvage.ts`

Therefore: **a rebuild + redeploy alone would already fix the visible 404.**

## Frontend coupling audit

`apps/web/src/lib/api.ts` profile() uses `/api/profiles/${encoded}` (correct).
No legacy unprefixed `/profiles/` calls from web source. The legacy unprefixed
*mount* in `apps/api/src/routes/index.ts` still exists for backward compat
(`registerAppRoutes(app)` mounted twice — once with `/api`, once bare).
Phase 7 removes it.

## Decisions taken for the fix

| Question | Decision |
|---|---|
| Crypto curve | secp256k1 (matches AT-proto) |
| KEK storage | `OPENXIV_KEK_BASE64` env var (KMS deferred until >100 users) |
| Legacy mount | Remove. Bare `/profiles/*`, `/papers/*` etc → 410 Gone (except `/auth/*/callback` for IdP redirect URIs) |
| `did:web:openxiv.local` URLs | 301 → canonical (handle or new did) at request time + DB row migration 0025 |
| Bluesky DID resolution | did:plc:* native via `@atproto/identity`. did:web fallback only on >3s timeout. Forward-only; no caching of PDS endpoint past 1h |
| Account linking | Implemented but executed **manually** by owner post-deploy (no auto-promote) |
| Owner did:plc | Pre-reserve `did:plc:dzhzljg4peg765tpd2q63luc` in `reserved_dids` table during 0024 seed |

## State to migrate

  SET did = 'did:web:openxiv.net:u:orcid.0009-0003-6027-7837',
      legacy_dids = ARRAY['did:web:openxiv.local:orcid.0009-0003-6027-7837']

because they haven't submitted anything yet).
