# HTML Rendering Edge Cases - 2026-05-20

Scope: generic paper HTML post-processing for current and future preprints. The implementation lives in `apps/web/src/lib/html-postprocess.ts` with citation linking in `apps/web/src/lib/citation-resolver.ts`.

## Conversion Output

- Full HTML documents are reduced to `<body>` content before rendering inline.
- LaTeXML chrome is stripped locally: generated stylesheets, page wrappers, navigation, and footer/header classes are removed from the reader body.
- Paragraphs are not split by math post-processing. Inline math remains inside the surrounding prose.
- Existing author markup is preserved unless a post-process step needs a reader wrapper or stable anchor.

## Math

- LaTeXML TeX annotations are normalized before KaTeX/MathJax reads them.
- Delimiter forms such as `\bigl{[}`, `\bigr{]}`, and escaped brace variants are rewritten to KaTeX-safe forms.
- Comment-only TeX fragments inside annotations are removed.
- LaTeXML abstract titles such as `<h3 class="ltx_title_abstract">` are promoted to `<h2>` so imported paper bodies do not skip heading levels under the page title.
- LaTeXML paragraph titles such as `<h4 class="ltx_title_paragraph">` are treated as styled paragraph text rather than structural headings. This preserves run-in section labels without creating invalid `h2` to `h4` jumps.
- Display equations, LaTeXML equation tables, KaTeX display blocks, and inline overflow-prone math receive horizontal scroll behavior on mobile rather than causing page-level viewport overflow.
- KaTeX CSS remains normal on desktop but is preloaded and applied asynchronously on mobile to avoid blocking the paper title LCP.
- Client KaTeX renders visual HTML only for LaTeXML annotations. This keeps formulas visually stable while avoiding a second hidden MathML tree for every expression on dense papers.

## Citations

- Bibliography entries are linked deterministically without network lookup.
- Supported identifiers: `arXiv:2301.01234v2`, bare `2301.01234`, legacy `hep-th/9901001`, and DOI forms such as `10.1145/123.456`.
- Malformed identifiers, for example `arXiv:bad` or `10.`, are left as text.
- Existing anchors are not re-linked, which avoids nested anchors in imported HTML.
- In-text citation anchors are given stable IDs and compact backlink chips are added to bibliography entries. The backlink copy avoids repeating literal "back" labels in the visible page.
- Bibliography sections headed "References", "Bibliography", or "Works cited" are wrapped in a native `<details>` accordion. It is open by default so desktop reading is not disrupted, while mobile users get a touch-friendly collapsible summary.
- Hovering or focusing an in-text bibliography link highlights the target reference entry on both inline paper pages and `/read` pages.
- Mobile reference sections opt out of `content-visibility: auto` so long bibliographies keep their real height and cannot visually collide with later natural-flow panels.

## Tables And Figures

- Tables are wrapped in `.paper-table-wrap` unless already wrapped, so wide data tables scroll horizontally inside the article body.
- LaTeXML tables with only `<td>` cells get first-row column headers inferred as `<th scope="col">`; authored `<th>` markup is left unchanged.
- SVG figures generated by TikZ/LaTeXML drop fixed width/height attributes and gain a derived `viewBox` when possible.
- SVGs without explicit `preserveAspectRatio` receive `preserveAspectRatio="xMidYMid meet"` to avoid cropping.
- Raster figures receive `loading="lazy"` and `decoding="async"` if the converter did not already emit those attributes. Existing width/height attributes are preserved; the post-processor does not invent dimensions when the source did not provide them.
- Mobile figure rules keep images and SVGs within the viewport. Inline paper figures are keyboard- and tap-openable in a native dialog so readers can inspect dense TikZ, plots, or raster figures without page-level horizontal scroll.
- Figure zoom ignores KaTeX, figcaption, and table-internal images/SVGs so math glyphs inside captions or table cells cannot become focusable hidden controls.
- Extracted gallery thumbnails use decorative image alt text inside a labelled button. The visible caption is included in the button accessible name, avoiding duplicate image-alt announcements while keeping touch zoom discoverable.

## Mobile Accessibility And Performance

- Site header is the only sticky surface. Paper side panels, AI Usage, Trust Passport, endorsements, artifacts, and raw HTML remain in natural document flow.
- Trust Passport lanes use native `<details>` semantics, not incompatible ARIA `listitem` roles.
- Mobile-only dark-theme contrast tokens are raised for secondary and tertiary text without changing desktop theme tokens.
- The Fraunces wordmark font keeps the existing blocking desktop load path, but mobile uses preload/onload so the non-critical wordmark font does not block paper reading LCP.
- Explainer tabs use 44px minimum touch targets and wrap instead of overlapping dense paper references.

## Fixtures

- `apps/web/test/fixtures/latexml-varied-disciplines.html` covers a non-physics title, inline math in prose, repeated reference backlinks, a TikZ-like SVG, a table, modern arXiv, legacy arXiv, DOI, and malformed identifiers.
- The fixture is intentionally discipline-neutral so the renderer does not depend on hardcoded `math-ph`, `gr-qc`, or specific paper IDs.

## Known Limits

- This is not a sanitizer. Inputs are expected to come from the server-side conversion pipeline.
- External DOI/arXiv metadata enrichment is intentionally out of scope for inline rendering. Rendering should not block on Crossref, arXiv, or another citation service.
- Unknown LaTeX packages may still degrade in the upstream converter, but the web reader should keep the resulting text, tables, math annotations, and figures inside the viewport.
